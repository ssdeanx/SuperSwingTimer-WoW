## Slam (TBC)
| Property | Value |
|----------|-------|
| Spell IDs | `1464, 8820, 11604, 11605, 25241, 25242` |
| Behavior | **Pause/Extend** — NOT a next-attack queue |
| Mechanism | `ns.PAUSE_SWING_SPELLS[spellId] = true` → sets `ns.pauseSwingTime` |
| Resume | `HandleSpellcastSucceeded` calls `AdjustSwingTimesAfterPause(now)` which shifts both MH/OH `lastSwing` forward by the pause duration |
| Interrupt | `HandleSpellcastInterruptedOrFailed` calls `AdjustSwingTimesAfterPause(now)` when pause was active |

### Pause/extend math (State.lua:694-708)
```
AdjustSwingTimesAfterPause(now):
  offset = now - pauseSwingTime
  mh.lastSwing += offset
  oh.lastSwing += offset
  pauseSwingTime = nil
```
- This effectively "freezes" the swing bar during the Slam cast, then resumes from where it would have been

## Parry haste (State.lua:936-956)
### Exact math
```
remaining   = (lastSwing + duration) - now
floor       = 0.2 * duration           // never below 20% remaining
reduction   = 0.4 * duration           // reduce by 40% of full swing
newRemaining = max(remaining - reduction, floor)
lastSwing   = now + newRemaining - duration
```
- Triggered by CLEU `SWING_MISSED` with missType `"PARRY"` when player is the target
- Applied to MH slot only
- `/sit` and parry event detection also triggers `UNIT_ATTACK_SPEED` refresh

## Extra attack suppression (Windfury / Sword Spec)
- `ns.extraAttackPending` counter in State.lua
- Incremented by CLEU `SPELL_EXTRA_ATTACKS`
- Each extra attack decrements the counter instead of starting a new swing cycle
- Prevents double-counting white swings generated by extra attack procs
