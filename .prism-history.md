# Prism Constraint History

### 2026-07-12 — Buff icon positioning overhaul + Flurry merger (prism-full)
- **Maximized:** Anchor resolution across all 6 class CD/buff icon groups + 2 Flurry icons. Found Hunter is the only class using dynamic `GetDebuffAnchorBar()`; Flurry icons (Warrior/Shaman) use hardcoded 12px offset and never participate in the `GetDebuffStackOffset()` system.
- **Sacrificed:** X-axis centering semantics — "centered with actual bars" is a design preference, not a correctness bug. Recorded as user request.
- **Adversarial gain:** Flurry icon `SetFrameStrata("DIALOG")` renders on top of debuff bars when overlapping, making the overlap visually destructive. This was invisible in the analytical passes.
- **Resolution (Flurry):** Dedicated Flurry icon system removed (~100 lines). Flurry now tracked by unified CD/buff icon group for Warrior and Shaman.
- **Resolution (buff icon Y-position):** Root cause was shared-state corruption in `_debuffStackIconOffset` — `RestackDebuffBars` early-return paths (nil-reference-bar, alpha≤0 check) set the offset to nil, causing `GetDebuffStackOffset()` to return the default -8 fallback even when debuff bars were visible. Fixed by replacing `GetDebuffStackOffset()` with `ComputeBuffIconOffset(barList, referenceBar)` in all 6 classes — same synthetic arithmetic, zero shared mutable state, computed fresh each frame from the exact same bar list that `RestackDebuffBars` just positioned.
- **Shield Block:** Removed from debuff bar stack; now tracked as a CD/buff icon in the Warrior group (spellId=2565, label="SB").
